01 · The problem

Most LLM output is padding.

Every response arrives loaded with tokens that carry zero information. Before a single useful word, the AI has already burned 40+ tokens on pleasantries it was never asked for.

~8 tok "I'd be happy to help you with that"
~8 tok "Let me take a look at that for you"
~7 tok "The reason this is happening is because"
~8 tok "Let me know if you need anything else!"
~5 tok "That's a great question"
~7 tok "I would recommend that you consider"
Total wasted before a single useful word ~43 tokens — on every response, every session
02 · The difference

Normal is verbose.
Caveman is blunt.
Introvert is precise.

Caveman saves tokens by breaking language — dropped articles, missing verbs, sentence fragments. You get smaller output, but you re-read every line. Introvert removes only what adds no information. Same signal, no noise, readable on the first pass.

Explaining a React re-render
🗣️ Normal ~69 tokens

"The reason your React component is re-rendering is likely because you're creating a new object reference on each render cycle. When you pass an inline object as a prop, React's shallow comparison sees it as a different object every time, which triggers a re-render. I'd recommend using useMemo to memoize the object."

🪨 Caveman ~19 tokens

"New object ref each render. Inline object prop = new ref = re-render. Wrap in useMemo."

Reads like a stack trace. You parse it — you don't read it.

🤫 Introvert ~20 tokens

"The inline object prop creates a new reference each render, triggering re-comparison — wrap it in useMemo."

Same savings. Reads like a sentence a senior engineer wrote.

Debugging an auth bug
🗣️ Normal ~42 tokens

"Sure! I'd be happy to help you with that. The issue you're experiencing is most likely caused by your authentication middleware not properly validating the token expiry. Let me take a look and suggest a fix."

🪨 Caveman ~14 tokens

"Bug in auth middleware. Token expiry check use < not <=. Fix:"

"Token expiry check use `<`" — you get the gist. "Check use" isn't English and you know it.

🤫 Introvert ~17 tokens

"Auth middleware checks expiry with < instead of <= — change it to <=."

Cause, fix, done. One sentence you could paste directly into a PR comment.

After generating code
🗣️ Normal ~48 tokens

"Here's the implementation for the API endpoint you requested. I've included error handling and input validation. This handles POST requests to create a new user, validates input, checks for duplicates, and hashes the password. Let me know if you'd like me to add anything else!"

🪨 Caveman ~10 tokens

"POST /users. Validation, dedup, hash. Done."

A telegram. Sounds like a robot filing a report.

🤫 Introvert ~12 tokens

"POST /users endpoint with input validation, duplicate check, and bcrypt hashing."

One sentence you'd write yourself in a commit message.

"Should I use Redis here?"
🗣️ Normal ~40 tokens

"That's a great question. While there are several approaches you could take, I think the most effective solution would be to use Redis for this particular use case because it handles this kind of access pattern natively and efficiently."

🪨 Caveman ~8 tokens

"Yes. Redis. Handle access pattern native."

Anyone who has used Caveman has re-read a line like this. It works. It shouldn't need re-reading.

🤫 Introvert ~9 tokens

"Yes — Redis handles this access pattern natively."

Confident. Correct. Grammatically flawless.

04 · What it does

Every response, distilled.

✂️

No preamble, ever

"Sure, I'd be happy to help!" is gone. Every response starts with the answer.

🧠

Expert precision

Technical terms stay exact. Polymorphism stays polymorphism. Nothing is paraphrased or dumbed down.

📐

Correct grammar

Full sentences. Proper articles. Brevity through precision — not through breaking language.

🛡️

Safety override

Destructive operations and security warnings get full, unambiguous language — then immediately back to quiet.

🎚️

Two intensity levels

lite removes preamble and filler (~35% reduction). full also removes hedging and qualifiers (~70% reduction).

🔁

Session persistent

Active every turn. Does not drift back to verbosity mid-session. Off only when explicitly deactivated.
